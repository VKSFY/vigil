What it detects

  • PE / EXE files — LightGBM classifier on EMBER 2018 features (2381-dim vector covering byte/entropy histograms, section info, hashed imports, data directories)
  • PowerShell scripts — LightGBM classifier on 40 hand-crafted features (obfuscation indicators, dangerous cmdlets, entropy, suspicious encodings)
  • Office macros — olevba extraction + weighted rule scoring (auto-exec triggers, shell calls, downloader objects, base64/hex obfuscation)
  • PDFs — pdfminer structural parse + marker regex (/JavaScript, /Launch, /OpenAction, chained-filter streams)
  • Behavioral patterns — process-creation monitor flags Office → shell spawn chains, PowerShell -EncodedCommand, hidden-window PowerShell, Startup-folder writes

How it works

Point Vigil at a file and it figures out what kind of file it is, then runs the right checks for that type — a learned model for programs and PowerShell scripts, structural rules for Word documents and PDFs. Turn on the realtime monitor and it watches a folder of your choice (Downloads is the usual pick), scanning every new file the moment it appears. If anything looks dangerous, the file is moved into a quarantine folder and the verdict shows up in a small local dashboard in your browser. Nothing leaves your machine — there's no cloud, no telemetry, no account.

Get started

Three commands, assuming you already have Python 3.10+ on PATH:

  1. git clone https://github.com/VKSFY/vigil
cd vigil
pip install -r requirements.txt
  2. python scan.py path\to\suspicious.exe
  3. python -m antivirus --watch C:\Users\You\Downloads

Step 2 scans one file and prints the verdict. Step 3 starts the tray app + dashboard and watches the folder you point it at. Full walkthrough in the user guide.

FAQ

Will this slow my computer down?
No. The filesystem watcher is event-driven, so it does nothing until a file lands in the watched folder. Each scan takes about 2 ms of model time plus disk I/O. The behavior monitor polls process IDs cheaply and only fetches details for new PIDs, sitting around 0.1 % CPU on a typical machine.
What happens to quarantined files — are they deleted?
No. They're moved to quarantine/<sha256>.quar with a sidecar JSON that records where they came from, why they were flagged, and the model's confidence. The .quar rename is purely there to prevent accidental execution. Files stay until you Restore or Delete them from the dashboard.
Is this a replacement for Windows Defender?
No. Vigil has no kernel driver, no signed-definition update channel, and no cloud lookups. Defender is your real protection — Vigil is a complementary tool that gives you a second opinion (especially on macros, PowerShell, and process behavior) and a local audit trail you control. Run both.
A file I know is safe got flagged — what do I do?
Open the dashboard, hit Restore in the Quarantine panel, then submit a correction: python scan.py feedback path\to\file.ext clean. The PowerShell model re-trains from feedback once you've collected 50+ corrections and only replaces itself if F1 strictly improves on the held-out split.
How do I uninstall it?
Remove the autostart key: reg delete HKCU\Software\Microsoft\Windows\CurrentVersion\Run /v AVMonitor /f, delete the Start Menu shortcut at %APPDATA%\Microsoft\Windows\Start Menu\Programs\Vigil.lnk, then delete the vigil folder. No service is installed; everything lives inside the repo directory.
