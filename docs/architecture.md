# Architecture

The system is split into three independent concerns:

1. **Detection** вҖ” given a file, return a verdict (clean / malicious / skipped), a confidence score, and a list of reasons. Stateless.
2. **Real-time monitoring** вҖ” watch the filesystem and process tree for events that should trigger detection, and react (log, quarantine, alert). Stateful per session.
3. **Distribution / lifecycle** вҖ” install, run, update model weights, collect user feedback, retrain.

Everything else is plumbing.

Each `_scan_*` lazy-loads its model on first call and caches the booster. A single `Scanner` instance handles thousands of scans without paying the ~100 ms LightGBM load cost per file.

### PE pipeline

`src/features_ember.py` is a vendored copy of EMBER's `features.py` patched for LIEF 0.14. It produces the exact 2381-dim float32 vector the EMBER 2018 dataset was trained on:

| Group | Dims | Source |
|---|---|---|
| ByteHistogram | 256 | Per-byte frequency over the whole file |
| ByteEntropyHistogram | 256 | 16Г—16 (window entropy, byte high-nibble) joint histogram |
| StringExtractor | 104 | Printable-string stats, char distribution, path/URL/registry/MZ counts |
| GeneralFileInfo | 10 | Size, virtual size, has_debug, exports, imports, has_signature, etc. |
| HeaderFileInfo | 62 | COFF + optional header fields + FeatureHasher buckets for string fields |
| SectionInfo | 255 | 5 general stats + 5 FeatureHasher buckets over section name Г— {size, entropy, vsize, entry, characteristics} |
| ImportsInfo | 1280 | 256-bin lib-name hash + 1024-bin "lib:func" hash |
| ExportsInfo | 128 | 128-bin exported-name hash |
| DataDirectories | 30 | 15 standard data dirs Г— (size, vaddr) |
| **Total** | **2381** | |

`PEFeatureExtractor.feature_vector(bytez)` returns the vector. `scan_api.Scanner._scan_pe` feeds it to LightGBM, then uses LightGBM's `predict(pred_contrib=True)` (TreeSHAP) to rank features by contribution, then maps flat indices back to human names via `features_ember.feature_name(idx)`.

### PowerShell pipeline

`src/ps1_extractor.py` defines 40 numeric features in a fixed order: shape (length, line count, entropy, comment ratio), obfuscation (base64 run length, hex literals, backticks, format strings, reversed strings), and dangerous primitives (IEX, DownloadString, AMSI bypass, Mimikatz mentions, Run key writes, scheduled tasks, WMI persistence). The model is a small LightGBM trained on 1000 synthetic scripts generated by `src/ps1_corpus.py`.

### Office (VBA) pipeline

`src/office_scanner.py` calls `oletools.olevba.VBA_Parser` on `.doc / .docm / .docx / .xls / .xlsm / .xlsx / .ppt / .pptm` and on raw VBA modules (`.bas / .cls / .frm`). The parser's `analyze_macros()` returns a list of `(category, keyword, description)` triples. The scorer applies a base weight per category (AutoExec=15, Suspicious=10, IOC=5, Base64=4, Dridex=8) and per-keyword bumps for the worst offenders (`URLDownloadToFile`=+12, `powershell`=+12, `Adodb.Stream`=+10). Threshold: 50.

### PDF pipeline

`src/pdf_scanner.py` runs two passes: a `pdfminer.six` structural parse to count pages and objects, and a regex pass over the raw bytes to find trigger keywords. Score weights:

| Marker | Weight |
|---|---|
| `/Launch` | 30 |
| `/JavaScript`, `/JS` | 25 each |
| `/OpenAction` | 20 |
| `/AA`, `/SubmitForm`, `/EmbeddedFile` | 15 each |
| `/RichMedia`, `/XFA` | 10 each |
| FlateDecode + ASCIIHexDecode chained | 20 |
| Object/page ratio вүҘ 50 | 10 |

Threshold: 35.

### `DirectoryWatcher`

Calls `ReadDirectoryChangesW` with `FILE_FLAG_OVERLAPPED` and waits on the overlapped event in 200 ms slices, polling a stop flag between waits. Stop is signaled by closing the directory handle (older pywin32 lacks `CancelIoEx`). A `_Debouncer` collapses bursty events into one scan call 500 ms after the last event for that path. The exclude list is checked against absolute paths, so the watcher never scans `quarantine/` or `logs/` even when the watch root is the repo root.

### `ProcessMonitor`

Two-phase polling. Phase 1: `psutil.pids()` returns a flat int list and we diff it against `_seen` вҖ” no per-process attribute resolution. Phase 2: for each new PID, fetch full attributes via `psutil.Process(pid)`. The original single-phase implementation called `process_iter(["username", ...])` per cycle, which triggered `LookupAccountSid` for every running process on every poll and starved the stop check.

### `SequenceDetector`

Keeps a `{pid вҶ’ ProcessEvent}` cache with 10-minute TTL so a child's rule can look up its parent's name even after the parent has exited (which is what makes the Office вҶ’ shell rule actually work вҖ” the spawned `cmd.exe` sticks around long after a renamed `winword.exe` exits).

`feedback.jsonl` rows carry the feature vector inline, so retraining doesn't have to re-extract from the original file (which may have moved or been deleted). Each row is one JSON line: timestamp, original path, file type, model verdict, user-supplied correct label, confidence, feature vector.

`changelog.json` records every retrain attempt вҖ” accepted *and* rejected. Rejected attempts increment the version counter but the archived model is the last *accepted* version (`_last_accepted_version()` in `src/retrain.py`). The archive keeps the most recent 5 accepted versions; older ones are GC'd.

The model auto-updater verifies SHA256 both before adopting a download (compare manifest claim against on-disk file вҶ’ decide whether to update) and after writing it (compute SHA256 of the downloaded temp file вҶ’ reject if mismatched, delete temp). On success, `os.replace` does the atomic rename.

## Why not вҖҰ

- **Why not signature-based AV?** Modern Windows malware has > 10^6 distinct hashes per family. Hash lists don't generalize. EMBER's feature set captures structure that hashes destroy.
- **Why not ETW for processes?** It's the right answer for sub-150ms processes and for injection detection, but it needs admin and a non-trivial C extension. psutil polling covers the in-scope rules without either.
- **Why not Sysmon?** Same answer plus a dependency on a separately-installed sensor. The whole point was a self-contained Python build.
- **Why local Flask, not Electron / Tauri?** Flask is 30 lines, ships in pip, and the dashboard is HTML вҖ” no reason to add a binary frontend.
- **Why not just use Defender?** This was built as a learning project to understand exactly what the components of an antivirus are. Defender is, of course, a much better antivirus.
